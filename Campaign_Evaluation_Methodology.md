# キャンペーン期間ごとの最大値取得と詰まり本数の評価方法

本プロジェクトにおける熱交換器の「詰まり傾向」の評価は、プラントの連続運転期間（キャンペーン）を基準として行われます。具体的には、各キャンペーンの「初期（Beginning）」と「末期（End）」のそれぞれ1か月間からパラメータの最大値を取得し、対応する期間の「詰まり本数」と照らし合わせるというプロセスを採用しています。

本ドキュメントでは、この評価アプローチの考え方や、抽出プロセスの詳細について解説します。他のプロジェクトでも同様のアプローチを応用する際の参考にしてください。

## 1. 考え方・背景

プラントの運転（キャンペーン）が進むにつれて、熱交換器内部のファウリング（汚れや詰まり）が進行し、伝熱性能（U値）の低下や圧力損失（ポンプ圧やジャケット圧の上昇）などの変化が生じます。

キャンペーン全体を一つのデータ点として扱うと、運転初期と末期での状態の変化（劣化の進行度合い）を捉えることができません。そこで、**キャンペーンを「初期（前半）」と「末期（後半）」に分割**し、それぞれの期間における状態の悪化を最も顕著に表す「最大値」を抽出して評価するアプローチを採用しています。

## 2. パラメータの最大値抽出プロセスと関連パラメータの取得

各キャンペーン期間から、以下のルールで「初期」と「末期」の期間を定義し、それぞれの期間内におけるパラメータ（例: UA、ジャケット圧、ポンプ圧）の最大値を抽出します。

### 対象期間の定義
- **初期（Beginning）:** キャンペーンの開始日から1か月間
- **末期（End）:** キャンペーンの終了日の1か月前から終了日まで

### 抽出するデータ（最大値とその発生日時、同時期の他パラメータ）
単に期間内の「最大値」を数値として取得するだけでは、異常の原因分析を行うのに不十分な場合があります。そのため、他のプロジェクトに応用する際も、**「最大値が発生した正確な日時（タイムスタンプ）」**と、**「その瞬間の他の関連パラメータ（温度、流量、圧力など）」**をセットで取得・記録することが極めて重要です。

#### なぜ日時と他のパラメータが必要なのか？
- **異常の要因特定:** あるパラメータ（例: 圧力）が最大になった瞬間に、別のパラメータ（例: 流量）がどのような状態だったかを確認することで、最大値が「実際の詰まり（設備劣化）」によるものか、「一時的な運転条件の変更（流量増など）」によるものかを判別できます。
- **データクレンジング:** センサーのノイズや誤作動による突発的な異常値（スパイク）を、他パラメータの挙動と照らし合わせて除外する際に役立ちます。

#### 抽出のプロセスイメージ
1. キャンペーン期間（`Period`）ごとにデータをグループ化する。
2. 「初期」または「末期」の条件に合致する1か月間のデータに絞り込む。
3. その期間内で目的のパラメータ（例: ジャケット圧）が最大となる行（レコード）を特定する。
4. その行の**「最大値」「タイムスタンプ」「その他の全パラメータ値」**を抽出する。
5. 抽出期間の基準日（例: キャンペーン開始日・終了日）を記録し、後続の分析に用いる。

## 3. 詰まり本数（NoS）の割り当てプロセス

設備の開放点検時などに確認される「詰まり本数（NoS: Number of Stuck tubes）」データは、キャンペーンの「前半（b: beginning）」と「後半（e: end）」に分けて記録されています。

これを、先ほど抽出したパラメータの最大値データと結びつけるために、共通の「結合キー」を作成します。

### 結合キーの作成ルール
キャンペーン期間の識別子（例: `2103`）と、期間の区分（`Beginning` または `End`）を結合した文字列をキー（`PeriodType`）とします。

- **パラメータ側（時系列データ）:**
  - `Period`（例: 2103） + `Type`（Beginning/End） ➔ **`2103_Beginning`** / **`2103_End`**
- **詰まり本数データ側（点検結果データ）:**
  - `Period`（例: 2103） + `b/e`フラグ ➔ **`2103_b`** または **`2103_e`**
  - ※ データ結合前に、フラグの表記を統一（例: `b` ➔ `Beginning`, `e` ➔ `End`）し、パラメータ側と同じ形式の `PeriodType` を作成します。

### データの結合と評価
作成した結合キー（`PeriodType`）を用いて、パラメータの最大値を含むレコードと、詰まり本数データを結合（JOIN）します。
これにより、「あるキャンペーンの前半（または後半）において最も状態が悪化した時のデータ」と、「その期間に発生した詰まり本数」を1行のデータとして対応付けることができ、回帰分析や相関の可視化が可能になります。

## 4. 他プロジェクトへの応用に向けてのまとめ

この手法を他の設備やプロセスの監視に応用する場合のポイントは以下の通りです。

1. **期間のセグメンテーション:**
   長期的な運転期間を、劣化が始まる「初期」と劣化が進行した「末期」（あるいは前半・後半）に分割することで、状態変化のコントラストを明確にする。
2. **最大値発生時のコンテキスト（文脈）の保存:**
   最大値という単一の数値だけでなく、**「いつ発生したか（タイムスタンプ）」**と**「その時、他のパラメータはどうだったか」**をセットで抽出・保持するプロセスを設計する。これにより、相関分析だけでなく、要因の深掘り（ドリルダウン）が可能になる。
3. **異なる粒度のデータを結ぶキー設計:**
   「連続的なセンサーデータ」から抽出した特徴量と、「定期点検ごとの結果データ」を結びつけるための明確な結合キー（キャンペーンID ＋ 前半/後半フラグ）をあらかじめ定義しておく。
